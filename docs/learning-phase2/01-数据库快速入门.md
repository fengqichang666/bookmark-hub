# Day 1-2: 数据库快速入门

> 前端类比：数据库 ≈ 持久化的JSON存储 + 强大的查询能力

## 1. 数据库基础概念

### 前端对比
```javascript
// 前端：数据存在内存中，刷新就丢失
let bookmarks = [
  { id: 1, title: "Google", url: "https://google.com" },
  { id: 2, title: "GitHub", url: "https://github.com" }
];

// 或者用localStorage（容量小，只能存字符串）
localStorage.setItem('bookmarks', JSON.stringify(bookmarks));
```

### 数据库：持久化存储
```sql
-- 数据库：数据永久保存在磁盘
-- 支持复杂查询、事务、并发控制

CREATE TABLE bookmarks (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(255) NOT NULL,
    url VARCHAR(500) NOT NULL,
    category VARCHAR(100),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

INSERT INTO bookmarks (title, url, category) 
VALUES ('Google', 'https://google.com', '搜索引擎');
```

### 为什么需要数据库？
- ✅ **持久化**：数据不会因程序重启而丢失
- ✅ **并发**：多用户同时访问
- ✅ **查询**：复杂的搜索、排序、聚合
- ✅ **事务**：保证数据一致性
- ✅ **容量**：可存储海量数据

## 2. 关系型数据库基础

### 核心概念

| 数据库概念 | 前端类比 | 说明 |
|-----------|---------|------|
| **数据库 (Database)** | 项目 | 一个应用的所有数据 |
| **表 (Table)** | 数组 | 存储同类数据 |
| **行 (Row)** | 对象 | 一条记录 |
| **列 (Column)** | 对象属性 | 字段 |
| **主键 (Primary Key)** | id | 唯一标识 |
| **外键 (Foreign Key)** | 引用 | 关联其他表 |

### 示例对比
```javascript
// 前端：数组 + 对象
const bookmarks = [
  { id: 1, title: "Google", url: "https://google.com", userId: 10 },
  { id: 2, title: "GitHub", url: "https://github.com", userId: 10 }
];

const users = [
  { id: 10, name: "Alice", email: "alice@example.com" }
];
```

```sql
-- 数据库：表 + 行
-- 用户表
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100),
    email VARCHAR(255)
);

-- 书签表（外键关联用户）
CREATE TABLE bookmarks (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(255),
    url VARCHAR(500),
    user_id BIGINT,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

## 3. SQL基础语法

### 3.1 创建表 (CREATE TABLE)

```sql
-- 创建书签表
CREATE TABLE bookmarks (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,      -- 主键，自增
    title VARCHAR(255) NOT NULL,               -- 标题，不能为空
    url VARCHAR(500) NOT NULL,                 -- URL
    description TEXT,                          -- 描述（长文本）
    category VARCHAR(100) DEFAULT '未分类',    -- 分类，默认值
    is_favorite BOOLEAN DEFAULT FALSE,         -- 是否收藏
    visit_count INT DEFAULT 0,                 -- 访问次数
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,  -- 创建时间
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 常用数据类型

| SQL类型 | Java类型 | 说明 | 示例 |
|---------|---------|------|------|
| `BIGINT` | `Long` | 长整数 | 用户ID、书签ID |
| `INT` | `Integer` | 整数 | 年龄、数量 |
| `VARCHAR(n)` | `String` | 可变长字符串 | 标题、URL |
| `TEXT` | `String` | 长文本 | 描述、内容 |
| `BOOLEAN` | `Boolean` | 布尔值 | 是否收藏 |
| `DECIMAL(10,2)` | `BigDecimal` | 精确小数 | 价格、金额 |
| `DATE` | `LocalDate` | 日期 | 生日 |
| `TIMESTAMP` | `LocalDateTime` | 日期时间 | 创建时间 |

### 3.2 插入数据 (INSERT)

```sql
-- 插入单条记录
INSERT INTO bookmarks (title, url, category) 
VALUES ('Google', 'https://google.com', '搜索引擎');

-- 插入多条记录
INSERT INTO bookmarks (title, url, category) VALUES
    ('GitHub', 'https://github.com', '开发工具'),
    ('StackOverflow', 'https://stackoverflow.com', '开发工具'),
    ('MDN', 'https://developer.mozilla.org', '文档');

-- 前端类比
bookmarks.push({ title: "Google", url: "https://google.com" });
```

### 3.3 查询数据 (SELECT)

```sql
-- 查询所有
SELECT * FROM bookmarks;

-- 查询指定列
SELECT id, title, url FROM bookmarks;

-- 条件查询（WHERE）
SELECT * FROM bookmarks WHERE category = '开发工具';

-- 模糊查询（LIKE）
SELECT * FROM bookmarks WHERE title LIKE '%Git%';

-- 多条件查询（AND/OR）
SELECT * FROM bookmarks 
WHERE category = '开发工具' AND is_favorite = TRUE;

-- 排序（ORDER BY）
SELECT * FROM bookmarks ORDER BY created_at DESC;

-- 限制数量（LIMIT）
SELECT * FROM bookmarks LIMIT 10;

-- 分页（LIMIT + OFFSET）
SELECT * FROM bookmarks LIMIT 10 OFFSET 20;  -- 第3页，每页10条
```

### 前端对比
```javascript
// 查询所有
bookmarks

// 条件查询
bookmarks.filter(b => b.category === '开发工具')

// 模糊查询
bookmarks.filter(b => b.title.includes('Git'))

// 排序
bookmarks.sort((a, b) => b.createdAt - a.createdAt)

// 限制数量
bookmarks.slice(0, 10)

// 分页
bookmarks.slice(20, 30)  // 第3页
```

### 3.4 更新数据 (UPDATE)

```sql
-- 更新单条记录
UPDATE bookmarks 
SET title = 'Google搜索', category = '搜索' 
WHERE id = 1;

-- 更新多条记录
UPDATE bookmarks 
SET is_favorite = TRUE 
WHERE category = '开发工具';

-- 增加访问次数
UPDATE bookmarks 
SET visit_count = visit_count + 1 
WHERE id = 1;

-- ⚠️ 注意：不加WHERE会更新所有记录！
UPDATE bookmarks SET is_favorite = TRUE;  -- 危险！
```

### 前端对比
```javascript
// 更新单条
const bookmark = bookmarks.find(b => b.id === 1);
bookmark.title = 'Google搜索';

// 更新多条
bookmarks
  .filter(b => b.category === '开发工具')
  .forEach(b => b.isFavorite = true);
```

### 3.5 删除数据 (DELETE)

```sql
-- 删除单条记录
DELETE FROM bookmarks WHERE id = 1;

-- 删除多条记录
DELETE FROM bookmarks WHERE category = '测试';

-- 删除所有记录（危险！）
DELETE FROM bookmarks;

-- 清空表并重置自增ID
TRUNCATE TABLE bookmarks;
```

### 前端对比
```javascript
// 删除单条
bookmarks = bookmarks.filter(b => b.id !== 1);

// 删除多条
bookmarks = bookmarks.filter(b => b.category !== '测试');

// 清空
bookmarks = [];
```

## 4. 聚合查询

```sql
-- 统计总数
SELECT COUNT(*) FROM bookmarks;

-- 按分类统计
SELECT category, COUNT(*) as count 
FROM bookmarks 
GROUP BY category;

-- 统计每个分类的平均访问次数
SELECT category, AVG(visit_count) as avg_visits 
FROM bookmarks 
GROUP BY category;

-- 找出访问最多的书签
SELECT * FROM bookmarks 
ORDER BY visit_count DESC 
LIMIT 1;

-- 统计收藏的书签数量
SELECT COUNT(*) FROM bookmarks WHERE is_favorite = TRUE;
```

### 前端对比
```javascript
// 统计总数
bookmarks.length

// 按分类统计
const countByCategory = bookmarks.reduce((acc, b) => {
  acc[b.category] = (acc[b.category] || 0) + 1;
  return acc;
}, {});

// 找出访问最多的
bookmarks.sort((a, b) => b.visitCount - a.visitCount)[0]
```

## 5. 连接查询 (JOIN)

```sql
-- 创建用户表
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(100) NOT NULL,
    email VARCHAR(255) NOT NULL
);

-- 书签表添加用户ID
ALTER TABLE bookmarks ADD COLUMN user_id BIGINT;
ALTER TABLE bookmarks ADD FOREIGN KEY (user_id) REFERENCES users(id);

-- 内连接：查询书签及其所属用户
SELECT b.title, b.url, u.username 
FROM bookmarks b
INNER JOIN users u ON b.user_id = u.id;

-- 左连接：查询所有书签，包括没有用户的
SELECT b.title, b.url, u.username 
FROM bookmarks b
LEFT JOIN users u ON b.user_id = u.id;

-- 查询某用户的所有书签
SELECT * FROM bookmarks 
WHERE user_id = (SELECT id FROM users WHERE username = 'alice');
```

### 前端对比
```javascript
// 内连接
bookmarks
  .filter(b => users.some(u => u.id === b.userId))
  .map(b => ({
    ...b,
    username: users.find(u => u.id === b.userId).name
  }));

// 左连接
bookmarks.map(b => ({
  ...b,
  username: users.find(u => u.id === b.userId)?.name || null
}));
```

## 6. 实战：H2内存数据库

### 为什么先用H2？
- ✅ 无需安装，嵌入式数据库
- ✅ 快速上手，适合学习
- ✅ 语法与MySQL兼容
- ✅ 自动创建表结构

### 在Spring Boot中使用H2

**pom.xml添加依赖：**
```xml
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>runtime</scope>
</dependency>
```

**application.yml配置：**
```yaml
spring:
  datasource:
    url: jdbc:h2:mem:bookmarkdb  # 内存数据库
    driver-class-name: org.h2.Driver
    username: sa
    password: 
  
  h2:
    console:
      enabled: true  # 启用H2控制台
      path: /h2-console  # 访问路径
  
  jpa:
    show-sql: true  # 显示SQL语句
    hibernate:
      ddl-auto: create-drop  # 自动创建表
```

**访问H2控制台：**
1. 启动Spring Boot应用
2. 浏览器访问：http://localhost:8080/h2-console
3. JDBC URL: `jdbc:h2:mem:bookmarkdb`
4. 用户名: `sa`，密码: 空

## 8. 常见问题

### Q1: VARCHAR(255)中的255是什么意思？
**A:** 最大字符数（不是字节数）：
```sql
VARCHAR(255)  -- 最多255个字符
VARCHAR(500)  -- 最多500个字符
TEXT          -- 无限制（最大65535字节）
```

### Q2: 什么时候用INT，什么时候用BIGINT？
**A:**
- `INT`: -21亿 ~ 21亿，适合数量、年龄等
- `BIGINT`: -900万亿 ~ 900万亿，适合ID、时间戳

### Q3: TIMESTAMP和DATETIME有什么区别？
**A:**
- `TIMESTAMP`: 存储UTC时间，自动转换时区，范围1970-2038
- `DATETIME`: 存储原始时间，不转换时区，范围1000-9999

推荐用`TIMESTAMP`。

### Q4: 为什么要用AUTO_INCREMENT？
**A:** 自动生成唯一ID：
```sql
-- 有AUTO_INCREMENT
INSERT INTO bookmarks (title, url) VALUES ('Google', 'https://google.com');
-- id自动为1

INSERT INTO bookmarks (title, url) VALUES ('GitHub', 'https://github.com');
-- id自动为2
```
